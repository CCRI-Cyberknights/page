👤 User & Role Management

Command Purpose Example & Notes
whoami Displays the effective username you are currently logged in as in the terminal. 
whoami
→ sandbox
su [user] Switch User. Changes the user context for the current terminal session. 
su bob
→ Prompts for bob's password, then switches to bob.
sudo Super User Do. Runs the subsequent command with root (administrator) privileges. 
sudo adduser new_user
→ Requires your password for authentication.
adduser [user] Creates a new user, including creating a home directory and group. Requires `sudo`. 
sudo adduser bob
→ Creates user bob, prompts for password and user info.

🔐 File Permissions (`chmod`)

Viewing Permissions: ls -l

The ls -l (long list format) command displays permissions as a 10-character string, followed by the owner and group.

drwxrw-r--  1  sandbox  sandbox  4096  Oct  12  20:00  my_directory
  • First character (File Type): d for directory, - for a regular file.
  • Next 3 (Owner): Permissions for the file owner.
  • Next 3 (Group): Permissions for users in the file's group.
  • Last 3 (Everyone Else): Permissions for all other users.

Permissions Breakdown (The Math)

Permissions are set using a three-digit number where each digit represents the Owner, Group, and Everyone Else, respectively. The digits are derived from summing values:

  • read = 4
  • write = 2
  • eXecute = 1
  • None = 0

Example Permission Math:
`rwx` (Read+Write+Execute) = 4 + 2 + 1 = 7
`rw-` (Read+Write+None) = 4 + 2 + 0 = 6
`r--` (Read+None+None) = 4 + 0 + 0 = 4

Command Purpose Example & Effect
chmod CHange MODe. Changes the read/write/execute permissions on a file or directory. 
chmod 666 file.txt
→ R/W for Owner/Group/Other (6+6+6)
chmod 700 private_folder
→ R/W/X for Owner only (7+0+0). Locks everyone else out.
chown CHange OWNer. Changes the owner of a file or directory. Requires `sudo`. 
sudo chown bob newfile.txt
→ Makes bob the new owner.

🔑 Key Takeaways

  • The Owner of a file (the user who created it) has the power to change its permissions with chmod.
  • A non-owner user, like bob, cannot change permissions using chmod without sudo.
  • A user must be explicitly added to the sudoers file to be able to use the sudo command. A newly created user typically does not have this privilege by default.
  • Root (administrator) users can override file ownership and change any file's permissions or contents, even if they don't own it. This is why sudo is extremely powerful.
  • Folder permissions (rwx on a directory) control the ability to list contents (r), create/delete files in it (w), and navigate into it (x). Restricting a folder can block a user from accessing their own files inside that folder.
