⌨️ Essential Commands

Command Purpose Example & Notes
rmdir Remove directory. Deletes an empty directory. 
rmdir folder_name
→ Works with absolute or relative paths
rmdir ~/desktop/james/rice
→ Example with full path
history Shows command history. Displays the last ~1000 commands. 
history
→ Shows numbered list of past commands
!12 Re-runs command number 12 from history. 
!12
→ Executes the 12th command from history
touch Creates empty files or updates file timestamps. 
touch example1
→ Creates empty file
touch example2 example3 ~/desktop/example4
→ Multiple files at once
cat Displays file contents. Different from ls which shows folder contents. 
cat filename
→ Shows file contents (not folder contents)
echo Prints text to terminal or redirects to files. 
echo "Hello World"
→ Prints to terminal
echo "Hello World" > example1
→ Overwrites file contents
>> Appends text to a file (keeps existing data). 
echo "Another line" >> example1
→ Adds to end of file without overwriting
cp Copies files or directories. 
cp file1 file2
→ Copies file
cp -r folder1 folder2
→ Copies directory recursively
mv Moves files or directories. Supports multiple files. 
mv example1 james/
→ Moves single file
mv example2 example3 ~/desktop/james/
→ Moves multiple files

📤 Redirection & Output

Understanding Redirection

Redirection operators allow you to control where command output goes:

  • > Overwrites a file (replaces all content)
  • >> Appends to a file (adds to existing content)
  • If the target file doesn't exist, both operators will create it

Practical Examples

ls / > example4
→ Saves directory listing to file 
history >> example4
→ Appends command history to same file 
echo "New file made this way" > example5
→ Creates new file with content 
cat file1 > file2
→ Copies file contents using redirection

⏳ Command History Navigation

Arrow Key Navigation

Use arrow keys to navigate through your command history:

  • ↑ Scrolls back through previous commands
  • ↓ Scrolls forward through command history
  • Press Enter to execute the selected command

History Number Execution

You can also execute commands by their history number:

history
→ Shows numbered list (e.g., 12 ls -la) 
!12
→ Executes command number 12

🔑 Key Takeaways

  • rmdir deletes empty directories.
  • Use history and !number to recall past commands quickly.
  • touch makes empty files; cat shows file contents.
  • > overwrites a file, >> appends to a file.
  • Redirects (>, >>) work with most commands, not just echo.
  • mv moves, cp copies — both support multiple files.
