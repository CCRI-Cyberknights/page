üì§ Redirection & Output

Understanding Redirection

Redirection operators allow you to control where command output goes:

  • > Overwrites a file (replaces all content)
  • >> Appends to a file (adds to existing content)
  • If the target file doesn't exist, both operators will create it

Practical Examples

ls / > example4
‚Üí Saves directory listing to file 
history >> example4
‚Üí Appends command history to same file 
echo "New file made this way" > example5
‚Üí Creates new file with content 
cat file1 > file2
‚Üí Copies file contents using redirection

üîë Key Takeaways

  • rmdir deletes empty directories.
  • Use history and !number to recall past commands quickly.
  • touch makes empty files; cat shows file contents.
  • > overwrites a file, >> appends to a file.
  • Redirects (>, >>) work with most commands, not just echo.
